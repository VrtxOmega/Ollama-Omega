A single-file MCP (Model Context Protocol) server that bridges **any Ollama model** — local or cloud — into your IDE as callable tools. Built to run alongside [Omega Brain MCP](https://github.com/VrtxOmega/omega-brain-mcp) in the VERITAS Omega ecosystem.

> **Origin Story:** This server was generated by **Qwen 3.5 397B** running on Ollama Cloud, then self-audited by the same 397B model, and hardened with 7 fixes applied by the Antigravity IDE agent. The cloud model built its own bridge into the IDE.

## Why This Exists

IDE model dropdowns are hardcoded. Extensions break across versions. Environment variables don't work. MCP does. This server gives your IDE direct access to:

- **Ollama Cloud** models (Qwen 3.5 397B, DeepSeek V3.1 671B, etc.) — no local GPU needed
- **Local** models (Llama, Gemma, Phi, Qwen, etc.) — pulled and running on your machine
- **Custom** models (fine-tuned, quantized, Modelfile-based) — anything Ollama can serve

## Tools (6)

| Tool | Ollama API | Purpose |
|------|-----------|---------|
| `ollama_health` | `GET /` + `GET /api/ps` | Check connectivity and list running models |
| `ollama_list_models` | `GET /api/tags` + `GET /api/ps` | List all models with loaded/unloaded status |
| `ollama_chat` | `POST /api/chat` | Chat completion with messages array |
| `ollama_generate` | `POST /api/generate` | Raw text generation for completions and code gen |
| `ollama_show_model` | `POST /api/show` | Model details: parameters, template, families, quantization |
| `ollama_pull_model` | `POST /api/pull` | Pull/download a model from the Ollama registry |

## Quick Start

### Requirements

- Python 3.11+
- [Ollama](https://ollama.com) running locally
- `pip install mcp httpx`

### Configure in Antigravity / Claude Desktop

```json
{
  "mcpServers": {
    "ollama": {
      "command": "python",
      "args": ["path/to/ollama_mcp_server.py"],
      "env": {
        "PYTHONUTF8": "1",
        "OLLAMA_HOST": "http://localhost:11434"
      }
    }
  }
}
```

### Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `OLLAMA_HOST` | `http://localhost:11434` | Ollama API base URL |
| `OLLAMA_TIMEOUT` | `300` | Request timeout in seconds (cloud models need 60-180s) |

### Using Cloud Models

Ollama Cloud models run on remote infrastructure — no local GPU required. After authenticating:

```bash
ollama login
```

You can call any cloud model directly:
```
ollama_chat(model="qwen3.5:397b-cloud", messages=[...])
```

Cloud models do **not** appear in `ollama_list_models` — they are accessed on-demand.

## Hardening

This server was self-audited by Qwen 3.5 397B and hardened with 7 fixes:

| # | Severity | Issue | Fix |
|---|----------|-------|-----|
| 2 | **HIGH** | `KeyError` crash on missing arguments | `_validate_required()` guard |
| 3 | **HIGH** | `resp.json()` crash on non-JSON responses | `_safe_json()` wrapper |
| 4 | **HIGH** | SSRF vector via `OLLAMA_HOST` env var | `follow_redirects=False` |
| 5 | **MEDIUM** | New HTTP client per call (no connection reuse) | Global singleton `httpx.AsyncClient` |
| 8 | **MEDIUM** | Zero logging/observability | Python `logging` to stderr |
| 9 | **LOW** | Duplicated payload construction (DRY violation) | `_build_options()` helper |
| 11 | **LOW** | Generic exception leaks internal paths | Sanitized error messages |
